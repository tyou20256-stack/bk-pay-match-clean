# BK P2P System — 機能仕様書

## 1. P2Pダッシュボード（index.html）

### 1.1 ヘッダー
- 暗号通貨切替ボタン（USDT / BTC / ETH）
- 最終更新タイマー（30秒カウントダウン）
- ダーク/ライトモード切替
- 日本語/英語切替

### 1.2 フィルターセクション
- **金額範囲フィルター:** スライダーで最小〜最大金額を指定
- **支払方法フィルター:** 銀行振込 / PayPay / LINE Pay 等（チップUI、「全て」で解除）
- **取引所フィルター:** Bybit / Binance / OKX（チップUI、「全て」で解除）
- **完了率フィルター:** 90%以上 / 95%以上 / 全て
- **オンラインフィルター:** オンラインのみ / 全て
- **ソート:** 価格順 / 数量順 / 完了率順

### 1.3 ヒーロセクション
- **購入 平均価格:** 全取引所の購入オーダーの平均価格
- **売却 平均価格:** 全取引所の売却オーダーの平均価格
- **平均スプレッド:** 平均売却 - 平均購入
- **ボリュームセクション:**
  - スポットレートから X% 以内の総売買可能額（JPY表示）
  - X は ±0.5% ステップで調整可能（デフォルト5%、範囲0.5〜20%）
- **乖離率フィルター:**
  - スポットレートから X% 以上乖離したオーダーを除外
  - ±1% ステップで調整可能（デフォルト15%、範囲1〜50%）

### 1.4 アービトラージバー
- ヘッダー下に全幅表示：「アービトラージ機会 N 件 発生中」
- クリックで展開 → 各機会の詳細（買い取引所/売り取引所/利益%/推定利益額）
- モバイルではボトムシート表示（70vh max）

### 1.5 オーダーブック
- 左: 購入オーダー一覧（安い順）
- 右: 売却オーダー一覧（高い順）
- 各オーダー表示項目: 価格、数量、金額範囲、支払方法、マーチャント名、完了率、取引所

### 1.6 レスポンシブ対応
- 768px以下: 1カラムレイアウト
- 480px以下: コンパクトフィルター、縮小テーブル

## 2. BK Pay（pay.html）

### 2.1 4ステップウィザード

**Step 1: 金額・方法選択**
- クイック金額ボタン（¥5,000 / ¥10,000 / ¥30,000 / ¥50,000 / ¥100,000 / ¥300,000 / ¥500,000 / ¥1,000,000）
- 自由入力フィールド（type=tel、iOSキーボード対応）
- 支払方法選択（銀行振込 / PayPay / LINE Pay / au PAY）
- リアルタイム換算表示（JPY → USDT）
- 「マッチング開始」ボタン（金額500円以上で有効化）

**Step 2: マッチング**
- 自動マッチング実行中のアニメーション
- 進捗ログ表示（「Bybit検索中...」「OKX検索中...」等）
- マッチ成功 → Step 3（AUTO MODE）
- マッチ失敗 → 「自社決済で続ける」ボタン → Step 3（SELF MODE）

**Step 3: 支払い**
- 振込先情報の表示
  - 銀行: 銀行名/支店名/口座種別/口座番号/口座名義/金額
  - 電子決済: QRコード画像 + 決済ID
- 「口座番号をコピー」ボタン
- 15分カウントダウンタイマー
  - 残り2分未満で赤色表示
  - 0分で自動キャンセル
- 「振込完了」ボタン / 「キャンセル」ボタン

**Step 4: 完了**
- 確認中表示 → 完了表示
- 注文サマリー（ID/モード/金額/USDT量/レート/取引所）

### 2.2 注文マッチングロジック

```
tryAutoMatch(amount, payMethod, crypto):
1. GET /api/rates/{crypto} で全取引所のオーダーを取得
2. フィルター条件:
   - payMethod が一致
   - minLimit <= amount <= maxLimit
   - merchant.completionRate >= 90%
   - merchant.isOnline === true
3. 該当あり → 最安値のオーダーを選択 → AUTO MODE
4. 該当なし → getAccountFromRouter() → SELF MODE
```

### 2.3 自社決済フォールバック

```
getAccountFromRouter(amount, payMethod):
1. POST http://localhost:3002/api/route にリクエスト
2. Account Routerが口座を選定（優先度/使用量/上限で最適化）
3. Router不可 → ハードコードされたフォールバック口座を使用
```

## 3. 管理画面（admin.html）

### 3.1 注文管理タブ
- 統計カード: 全注文数 / 支払待ち / 完了 / キャンセル
- 注文テーブル: ID/モード/金額/USDT/レート/取引所/支払方法/ステータス/日時
- 操作ボタン: 承認（confirming→completed）/ 却下（→cancelled）

### 3.2 銀行口座タブ
- 口座追加フォーム: 銀行名/支店名/口座種別/口座番号/名義/上限額/優先度/状態/メモ
- 口座一覧テーブル: 状態インジケーター（緑=稼働/黄=休止/赤=凍結）
- 操作: 稼働↔休止切替 / 削除

### 3.3 電子決済タブ
- PayPay: ID/表示名/QRコード画像アップロード
- LINE Pay: ID/送金リンクURL/QRコード画像アップロード
- au PAY: ID/表示名/QRコード画像アップロード

### 3.4 取引所APIタブ
- Bybit: メール/パスワード/APIキー/APIシークレット/2FA TOTPシークレット
- OKX: メール/パスワード/APIキー/APIシークレット/パスフレーズ/2FA
- Puppeteer自動取引ステータス表示

### 3.5 ウォレットタブ
- USDT受取アドレス（TRC-20）
- ラベル

### 3.6 設定タブ
- 最低完了率（%）/ 注文タイムアウト（分）
- 最低・最大注文額（円）
- オンラインのみマッチング（はい/いいえ）
- フォールバックモード（自社決済/注文拒否）

## 4. レート集約エンジン

### 4.1 更新サイクル
- 30秒間隔で全暗号通貨（USDT/BTC/ETH）のレートを更新
- 各取引所に並列リクエスト（Promise.all）

### 4.2 乖離率フィルター（サーバー側）
- CoinGeckoからスポットレートを取得
- 各オーダーの価格がスポットから±maxDeviationPct%を超える場合は除外
- フィルター結果はコンソールにログ出力

### 4.3 アービトラージ検出
- 全取引所ペアのクロス比較
- 利益率がarbitrageThreshold%以上の組み合わせを検出
- タイムスタンプ・持続時間・推定利益額を追跡
- 開始/終了をログ出力

## 5. 認証システム

### 5.1 ログインフロー
```
1. POST /api/auth/login {username, password}
2. SHA-256(password + 'bkpay-salt') でハッシュ比較
3. 一致 → 32バイトランダムトークン生成
4. sessionsテーブルにINSERT（24時間有効）
5. Set-Cookie: bkpay_token=<token>; HttpOnly
```

### 5.2 認証チェック
```
リクエスト受信
  → Cookie `bkpay_token` または Authorization ヘッダーからトークン取得
  → sessionsテーブルで検証（トークン一致 AND expires_at > 現在時刻）
  → 有効: 通過 / 無効: 401
```

### 5.3 保護対象
- `/admin.html` → 未認証は `/login.html` にリダイレクト
- `GET /api/orders` — 注文一覧
- `/api/accounts/*` — 口座管理
- `/api/epay/*` — 電子決済設定
- `/api/trader/*` — 取引所API設定
- `/api/wallet/*` — ウォレット設定
- `/api/settings/*` — システム設定
