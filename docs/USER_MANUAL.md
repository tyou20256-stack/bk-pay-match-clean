## 1. Pay Matchとは

Pay Matchは、P2P暗号通貨取引のレート比較・マッチング・決済プラットフォーム。

**主な機能:**
- 複数取引所（OKX・Bybit・Binance）のP2Pレートをリアルタイム比較
- JPY ↔ USDT/BTC/ETH の取引マッチング
- 銀行振込 → 暗号通貨受取の自動化
- Telegram Botによる注文・通知
- アービトラージ機会の検出

**対応言語:** 日本語・英語・中国語・ベトナム語

## 2. カスタマー向けガイド

### 2.1 アカウント登録

1. https://bkpay.app/customer-login.html を開く
2. 「新規登録」タブをクリック
3. 以下を入力:
   - メールアドレス
   - 表示名（任意）
   - パスワード（8文字以上）
   - パスワード確認
4. 「アカウント作成」をクリック
5. 「登録完了」メッセージ表示後、自動でログインタブに切り替わる

### 2.2 ログイン

1. https://bkpay.app/customer-login.html を開く
2. メールアドレスとパスワードを入力
3. 「ログイン」をクリック
4. カスタマーダッシュボードにリダイレクトされる

### 2.3 カスタマーダッシュボード

ログイン後、https://bkpay.app/customer-dashboard.html にアクセス可能。

**タブ構成:**

| タブ | 内容 |
|------|------|
| 概要 | JPY・USDT・BTC・ETH の残高表示、最近の取引5件 |
| 取引履歴 | 全取引の一覧（種類・通貨でフィルター可能、ページネーション付き） |
| KYC認証 | 本人確認書類のアップロード（パスポート・免許証・マイナンバーカード等） |
| プロフィール | 表示名・電話番号の編集 |

### 2.4 KYC認証の手順

1. カスタマーダッシュボード → 「KYC認証」タブ
2. 書類の種類を選択（パスポート / 運転免許証 / マイナンバーカード等）
3. 書類画像をドラッグ＆ドロップまたは選択してアップロード
   - 対応形式: JPG, PNG, PDF（最大10MB）
4. 「送信」をクリック
5. ステータスが「審査中」に変わる
6. 管理者が承認すると「認証済み」になる

**KYCステータス:**
- 未提出 → 審査中 → 認証済み / 却下

### 2.5 暗号通貨の購入フロー

```
1. 注文作成（金額・通貨を指定）
     ↓
2. 振込先の銀行口座情報が表示される
     ↓
3. 指定の銀行口座に日本円を振り込む
     ↓
4. 「支払い済み」ボタンをクリック
     ↓
5. 管理者（またはセラー）が入金を確認
     ↓
6. USDTが自動的にウォレットに送金される
     ↓
7. 取引完了
```

### 2.6 暗号通貨の売却フロー

```
1. 売却注文を作成（暗号通貨の数量を指定）
     ↓
2. 指定のウォレットアドレスに暗号通貨を送金
     ↓
3. 送金確認後、日本円が指定口座に振り込まれる
     ↓
4. 取引完了
```

## 3. 管理者向けガイド

### 3.1 管理者ログイン

1. https://bkpay.app/login.html を開く
2. ユーザー名とパスワードを入力
3. MFA（二要素認証）が有効な場合、6桁のTOTPコードを入力
4. ログイン後、管理画面にリダイレクトされる

### 3.2 MFA（二要素認証）の設定

**初回ログイン後すぐに設定を推奨。**

1. 管理画面 → 「ユーザー」タブ
2. 自分のアカウントの「MFA有効化」をクリック
3. QRコードが表示される
4. Google Authenticator等のアプリでスキャン
5. アプリに表示される6桁コードを入力して確認
6. 以後、ログイン時にTOTPコードが必須になる

### 3.3 管理画面タブ一覧

管理画面（https://bkpay.app/admin.html）には以下のタブがあります。

#### 注文管理

全注文の一覧と操作。

**注文ステータスフロー:**
```
新規(Pending) → 支払い済み(Paid) → 確認済み(Verified) → 送金完了(Completed)
                                                        → キャンセル(Cancelled)
                                                        → 期限切れ(Expired)
```

**操作ボタン:**
- 「支払い確認」— カスタマーの銀行振込を確認
- 「暗号通貨送金」— USDTをカスタマーのウォレットに送金
- 「手動完了」— 送金済みの注文を手動で完了にする
- 「キャンセル」— 注文を取り消す

#### 銀行口座

カスタマーが振り込む先の銀行口座を管理。

- 「口座追加」で新規登録（銀行名・支店・口座番号・名義・日次上限・メモ）
- ステータス: Active（稼働中）/ Rest（一時停止）
- CSVから一括インポート可能
- 口座の凍結リスクモニタリング

#### 電子決済

PayPay / LINE Pay / au PAY の設定。

#### 取引所API

Bybit・Binance・OKX・HTXのAPI接続設定。

- APIキー・シークレットの登録
- Puppeteerトレーダーのログインテスト
- ブラウザセッションのスクリーンショット確認

#### ウォレット

TRONウォレット（USDT送金用）の管理。

- 残高表示（USDT / TRX）
- ホット/コールドウォレットの閾値設定
- 手動スイープ（ホット→コールド移動）
- 暗号通貨トランザクション履歴

#### 設定

| 設定項目 | 説明 |
|----------|------|
| 最小完了率 | マーチャントの最低完了率フィルター |
| 注文タイムアウト | 注文の有効期限（分） |
| 最小/最大注文金額 | JPYでの注文上下限 |
| オンラインのみ | オンラインマーチャントのみ表示 |
| フォールバックモード | 取引所ダウン時の動作 |

#### スプレッド

通貨ごとのスプレッド設定と最適化提案。24時間統計も表示。

#### レポート

- **日次レポート**: 日付指定で注文数・金額・手数料を集計
- **月次レポート**: 月指定で月間サマリー
- **CSVエクスポート**: 会計ソフト連携（freee / 弥生形式対応）

#### 顧客管理

カスタマー一覧・紹介コード・取引量の管理。

#### 手数料

カスタマーランク別の手数料率設定と手数料レポート。

#### ユーザー（管理者アカウント）

- 管理者アカウントの追加・削除
- ロール: viewer（閲覧のみ）/ manager（操作可能）/ admin（全権限）
- パスワードリセット
- MFA有効化/無効化

#### 取引上限

- グローバル上限（全カスタマー共通）
- カスタマー別上限
- 決済方法別上限
- 種類: 1回あたり / 日次 / 週次 / 月次

#### APIキー

外部システム連携用のAPIキー発行・管理。

#### P2Pセラー

セラー申請の承認・管理。

#### 出金管理

カスタマーの出金リクエスト処理。

#### 自動取引

Puppeteerベースの自動売買Botの設定・状態確認。

### 3.4 データエクスポート

管理画面からCSVエクスポートが可能:

| エクスポート | 用途 |
|-------------|------|
| 注文一覧 | 取引記録 |
| freee形式 | freee会計ソフトにインポート |
| 弥生形式 | 弥生会計にインポート |
| 銀行口座一覧 | 口座管理 |
| 手数料レポート | 手数料集計 |

## 4. セラー向けガイド

### 4.1 セラー登録

1. https://bkpay.app/seller-register.html を開く
2. 以下を入力:
   - 氏名
   - メールアドレス
   - パスワード（8文字以上）
   - PayPay ID（任意）
   - LINE Pay ID（任意）
   - au PAY ID（任意）
   - 最小取引金額（JPY）
   - 最大取引金額（JPY）
3. 「登録申請」をクリック
4. 管理者の承認を待つ

### 4.2 セラーダッシュボード

https://bkpay.app/seller-dashboard.html でログイン後:

**表示内容:**
- 利用可能残高（USDT）
- ロック中残高（エスクロー中のUSDT）
- 完了した取引数
- ステータス

**注文テーブル:**
- 直近50件の注文一覧
- 注文ID・金額・USDT数量・決済方法・ステータス・日時

### 4.3 入金確認フロー

カスタマーが銀行振込を完了した後:

1. セラー確認URL（`/seller-confirm.html?orderId=XXX`）にアクセス
2. 注文詳細を確認（金額・暗号通貨数量・決済方法）
3. 「入金確認（USDTを送金する）」をクリック
4. USDTがカスタマーに自動送金される

## 5. Telegram Bot

### 5.1 Botの開始

1. Telegramで Bot にメッセージを送信
2. `/start` コマンドを送信
3. メニューボタンが表示される

### 5.2 コマンド一覧

| コマンド | 機能 |
|---------|------|
| `/start` | メニュー表示 |
| `/buy` | 暗号通貨の購入（JPY金額を指定） |
| `/sell` | 暗号通貨の売却（数量を指定） |
| `/rates` | 現在のレート確認（USDT/BTC/ETH） |
| `/status` | 注文ステータス確認（注文IDを指定） |
| `/history` | 注文履歴の表示 |
| `/calc` | 金額計算機（レートシミュレーション） |
| `/alert` | 価格アラートの設定 |
| `/notify` | 通知設定のON/OFF |
| `/help` | 使い方ガイド |
| `/lang` | 言語設定（JP/EN/ZH/VI） |

### 5.3 Botでの購入手順

```
1. /buy を送信
     ↓
2. クイック金額ボタンから選択（¥5,000 / ¥10,000 / ¥30,000 / ¥50,000 / ¥100,000）
   または金額を直接入力（例: 50000、5万、10万円）
     ↓
3. 注文確認メッセージが表示される（レート・数量・振込先）
     ↓
4. 指定口座に銀行振込
     ↓
5. 入金確認後、USDTが自動送金される
     ↓
6. 完了通知がBotに届く
```

### 5.4 メニューボタン

`/start` で表示されるクイックメニュー:

- 💰 USDT購入
- 💱 暗号通貨売却
- 📊 レート確認
- 🔍 注文確認
- 📋 注文履歴
- 🧮 金額計算
- ⏰ アラート設定
- 🔔 通知設定
- 📖 使い方ガイド
- 🌐 Pay Matchを開く
- 📊 マイページ

## 7. トラブルシューティング

### ログインできない

| 症状 | 対処 |
|------|------|
| パスワードが違う | 管理者にパスワードリセットを依頼 |
| ログイン画面にリダイレクトされる | ブラウザのCookieをクリアして再試行 |
| MFAコードが通らない | 端末の時刻が正確か確認（NTP同期） |

### 注文が完了しない

| 症状 | 対処 |
|------|------|
| 「支払い済み」にならない | 振込先口座・金額が正しいか確認 |
| USDTが届かない | 管理画面で注文ステータスを確認。ウォレット残高を確認 |
| 注文が期限切れ | 新しい注文を作成する |

### Telegram Bot が反応しない

| 症状 | 対処 |
|------|------|
| コマンドに応答なし | サーバーのDockerコンテナが稼働中か確認 |
| 通知が来ない | `/notify` コマンドで通知設定をONにする |
| 通知が多すぎる | 管理者に閾値の調整を依頼 |

### ページが表示されない

| 症状 | 対処 |
|------|------|
| 502 Bad Gateway | サーバーでDockerコンテナを再起動: `docker compose restart` |
| 接続タイムアウト | Cloudflare DNS設定とサーバー稼働状態を確認 |
| HTTPS エラー | Cloudflare SSL設定が「Full」になっているか確認 |

## 付録: API一覧（外部連携用）

### 認証

外部APIは `X-API-Key` ヘッダーで認証。管理画面の「APIキー」タブで発行。

### 主要エンドポイント

```
POST /api/v1/orders          — 買い注文作成
GET  /api/v1/orders/:id      — 注文ステータス確認
GET  /api/v1/balance         — ウォレット残高確認
```

### レート情報（認証不要）

```
GET  /api/rates              — 全通貨のレート
GET  /api/rates/:crypto      — 特定通貨のレート（USDT/BTC/ETH）
GET  /api/best               — ベストレート（買い・売り）
GET  /api/spread             — スプレッド比較
GET  /api/arbitrage          — アービトラージ機会
GET  /api/history/:crypto    — 価格履歴
```
