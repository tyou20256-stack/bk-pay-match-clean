# BK P2P System — 用語集・ビジネスコンテキスト

## 1. このシステムが必要な理由

### 背景
- オンラインサービスの決済で、顧客の日本円をUSDT（暗号通貨）に変換する必要がある
- 銀行口座は凍結リスクがあり、約3日に1回のペースで使用不能になる
- 約100口座を運用し、スマートに振り分ける仕組みが必要
- P2P取引所のレートは取引所ごとに異なり、最安レートで購入したい

### 解決策
- 3取引所のP2Pレートをリアルタイム比較 → 最安で購入
- 自動マッチング + 自社口座フォールバックの二段構え
- 口座ローテーションで凍結リスクを分散
- 将来的にPuppeteerで取引所操作を自動化

## 2. ビジネス用語

| 用語 | 説明 |
|------|------|
| P2P取引 | Peer-to-Peer。取引所を介さず個人間で暗号通貨を売買する方式。Bybit/Binance/OKX等が仲介プラットフォームを提供 |
| マーチャント | P2P取引所で暗号通貨を売買する業者/個人。完了率やオーダー数で信頼度が測れる |
| 完了率 | マーチャントの取引完了率。90%以上を信頼基準とする |
| スプレッド | 購入価格と売却価格の差。小さいほど市場効率が良い |
| アービトラージ | 取引所間の価格差を利用した裁定取引。A取引所で安く買い、B取引所で高く売る |
| スポットレート | CoinGecko等から取得する暗号通貨の市場基準価格 |
| 乖離率 | P2Pオーダーの価格がスポットレートからどれだけ離れているかの割合（%） |

## 3. システム用語

| 用語 | 説明 |
|------|------|
| AUTO MODE | P2P取引所でマッチングが成功したモード。取引所マーチャントの振込先を顧客に表示 |
| SELF MODE | マッチング失敗時のフォールバック。自社銀行口座を顧客に表示 |
| Account Router | 別プロセス（Port 3002）で動作する口座ローテーションシステム。最適な口座を自動選定 |
| Puppeteer Trader | ヘッドレスブラウザで取引所にログインし、注文を自動作成するフレームワーク。現在は構造のみで未稼働 |
| フォールバック | 主要な処理が失敗した場合の代替処理。AUTO失敗→SELF、Account Router不可→DB口座 |
| 乖離率フィルター | スポットレートから大きく離れた異常なオーダーを自動除外する機能 |
| ボリューム閾値 | スポットレートから指定%以内のオーダーの合計取引可能額 |

|--------|------------|---------|------|

## 6. 注文ステータス

| ステータス | 日本語 | 画面表示 | 説明 |
|-----------|-------|---------|------|
| matching | マッチング中 | 検索中 | 取引所でマッチング検索中 |
| pending_payment | 支払待ち | 支払待ち | 顧客に振込先を表示中。15分のタイマー |
| paid | 支払済み | 支払済 | 顧客が振込完了を報告 |
| confirming | 確認中 | 確認中 | 入金確認処理中 |
| completed | 完了 | 完了 | 取引完了。USDT送付済み |
| cancelled | キャンセル | キャンセル | 顧客または管理者がキャンセル |
| expired | 期限切れ | 期限切れ | 15分以内に振込がなかった |

---

## 7. 口座ステータス

| ステータス | 説明 | UI表示 |
|-----------|------|--------|
| active | 稼働中。注文の振込先として使用される | 🟢 稼働中 |
| rest | 休止中。手動で一時停止。振込先として使用されない | 🟡 休止中 |
| frozen | 凍結。銀行により口座が凍結された | 🔴 凍結 |

---

## 8. 口座優先度

| 優先度 | 説明 |
|-------|------|
| high | 最優先で使用。信頼性の高い口座 |
| medium | 通常使用 |
| low | 他の口座が使用上限に達した場合のみ使用 |

---

## 9. 設定値

| キー | デフォルト | 説明 |
|------|----------|------|
| minCompletion | 90 | マーチャントの最低完了率（%）。これ未満は除外 |
| orderTimeout | 15 | 注文の有効期限（分） |
| minAmount | 500 | 最低注文額（円） |
| maxAmount | 1000000 | 最大注文額（円） |
| onlineOnly | yes | オンラインのマーチャントのみマッチング |
| fallbackMode | self | マッチ失敗時: self=自社決済 / reject=注文拒否 |
