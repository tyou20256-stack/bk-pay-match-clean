1システム概要

BK Pay Match は、14のP2P取引所（Bybit, Binance, OKX, KuCoin, Gate.io, MEXC, Paxful, Noones, HodlHodl, Bisq, AgoraDesk, Peach, RoboSats, BK Pay自社）からリアルタイムでレートを収集し、顧客の暗号資産購入/売却注文を最適なレートでマッチングするシステムです。

アーキテクチャ

層技術説明
バックエンドNode.js + Express + TypeScriptREST API / WebSocket
データベースSQLite (better-sqlite3)注文・口座・設定・監査ログ
フロントエンドHTML/CSS/JS (Vanilla)管理画面 + 顧客ページ
暗号通貨TronWeb (TRC-20)USDT自動送金 + TX確認ポーリング
自動取引OKX C2C API / Puppeteer取引所への自動注文作成
通知Telegram Bot API注文・アラート通知
ログ構造化JSON Loggerリクエスト相関ID付きログ

主要ポート

ポート用途
3003 (デフォルト)HTTP / WebSocket

2環境構築・初回セットアップ

動作要件

  • Node.js 20 以上
  • npm 10 以上
  • Windows / Linux / macOS
  • 最小: 2 CPU / 4GB RAM / 10GB SSD

インストール手順

# 1. リポジトリクローン
git clone <repo-url> bk-pay-match
cd bk-pay-match

# 2. 依存パッケージインストール
npm ci

# 3. 環境変数設定
cp .env.example .env
# .env を編集して必要な値を設定

# 4. TypeScript ビルド
npm run build

# 5. 起動
npm start

.env 必須設定

変数必須説明
BK_ADMIN_PASSWORD必須管理者パスワード（初回ログイン用、必ず変更）
BK_ENC_KEY必須取引所APIキー暗号化用の32バイトHEXキー
PORT任意サーバーポート（デフォルト: 3003）
NODE_ENV任意development / production
TRON_WALLET_PRIVATE_KEY送金時USDT送金用TRONウォレット秘密鍵
TRONGRID_API_KEY送金時TronGrid APIキー
暗号化キー生成方法 node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
本番環境では必ず実行 初回ログイン後、管理画面「設定」からパスワードを変更してください。デフォルトパスワード（bkpay2026）を本番で使い続けないでください。

3サーバー起動・停止

起動方法

方法コマンド用途
直接起動npm startフォアグラウンド実行
開発モードnpm run devファイル変更で自動リロード
PM2npm run pm2:startデーモン化 + 自動再起動
Dockernpm run docker:upコンテナ化

停止方法

# 直接起動の場合: Ctrl+C（グレースフルシャットダウン対応）
# PM2
npm run pm2:stop

# Docker
npm run docker:down
グレースフルシャットダウン SIGTERM/SIGINT 受信時、進行中のリクエスト完了を待ってからプロセスを終了します。TX確認ポーリングやcronジョブも安全に停止されます。

管理画面へのアクセスにはログインが必要です。http://<host>:3003/login.html にアクセスしてください。

ログイン画面 — ユーザー名とパスワードを入力

デフォルト認証情報

項目値
ユーザー名admin
パスワードBK_ADMIN_PASSWORD 環境変数の値（未設定時: bkpay2026）

6管理画面：銀行口座

自社決済（SELFモード）で使用する銀行口座を管理します。顧客に振込先として表示されます。

銀行口座管理 — 口座一覧、状態・優先度・1日上限の管理

各項目の説明

項目説明
状態稼働中 / 休止。休止中の口座はマッチングに使われない
信頼度スコア緑バッジの数値（100が最大）。口座の信頼性指標
1日上限その口座で1日に受け付ける最大金額（円）
今日の使用額当日の累計使用額。上限に達すると自動的に使用停止
優先度high / medium / low — マッチング時の優先順位

操作

  • + 口座を追加: 新しい銀行口座を登録
  • CSV一括登録: CSV ファイルから複数口座を一括登録
  • 口座一覧CSV: 現在の口座一覧をCSVでエクスポート
  • 全口座ヘルスチェック: 全口座の稼働状況を一括確認
  • 休止 / 削除: 個別口座の停止・削除
口座情報は暗号化保存 銀行口座の名義・口座番号は BK_ENC_KEY で暗号化されてDBに保存されます。キーを紛失すると復号できません。

7管理画面：ウォレット

USDT (TRC-20) の受取・送金ウォレットを管理します。

ウォレット管理 — 受取アドレス設定、送金ウォレット状態、USDT送金履歴

受取ウォレット (TRC-20)

顧客がUSDTを送付する宛先アドレス。Tで始まるTRONアドレスを設定します。

アドレスを間違えると資金を失います 設定後は必ず少額のテスト送金で確認してください。

送金ウォレット (USDT自動送金)

注文完了時に顧客へUSDTを自動送金するウォレット。TRON_WALLET_PRIVATE_KEY 環境変数で設定します。

  • 秘密鍵が未設定の場合、「送金ウォレット: 未設定」と表示される
  • 送金にはウォレットに十分なUSDT残高 + TRX（ガス代）が必要
  • 送金完了後、TX IDがTRONブロックチェーン上で確認される（自動ポーリング）

USDT送金履歴

過去の送金記録（注文ID, 通貨, 数量, 送信先, TX ID, 日時）を確認できます。

銀行入金検証 (自動)

画面下部の「銀行入金検証」セクションで、銀行入金記録を登録し、confirming 状態の注文と自動照合できます。

11公開ページ：顧客決済

顧客が暗号資産を購入・売却するインターフェース。http://<host>:3003/pay.html

Pay Match — 顧客向け決済フロー（4ステップ）

決済フロー

  1. 金額・方法: 通貨（USDT/BTC/ETH）、金額（プリセットまたは自由入力）、支払方法（銀行振込/PayPay/LINE Pay/au PAY）を選択
  2. マッチング: 最適レートのP2Pセラーを自動検索
  3. お支払い: 表示された振込先に入金
  4. 完了: 入金確認後、USDTが自動送金される

購入/売却

  • 購入（円→暗号通貨）: 円を支払い、USDTを受け取る
  • 売却（暗号通貨→円）: USDTを送付し、円を受け取る
顧客への約束 エスクロー保護 / 5-15分で完了 / 手数料 2.0%

13ヘルスチェック・監視

ヘルスチェック API

GET /api/health

サーバーの稼働状態を返します。ロードバランサーやモニタリングツールのヘルスチェック先として使用。

ヘルスチェック — /api/health エンドポイント

ログ

すべてのログはJSON構造化形式で出力されます。各リクエストにはCorrelation ID（reqId）が付与され、一連のリクエスト処理をトレースできます。

// ログ出力例
{
  "level": "info",
  "message": "Order created",
  "orderId": "ORD-MMB135BM-P97N",
  "amount": 8000,
  "crypto": "USDT",
  "reqId": "abc-123-def",
  "timestamp": "2026-03-05T08:15:30.123Z"
}

監査ログ

管理者のすべての操作は audit_log テーブルに記録されます。

GET /api/admin/audit-log?action=login&limit=50

記録される操作: login, login_failed, logout, password_change, order_verify, order_send_crypto, order_manual_complete

メトリクス

GET /api/metrics

Prometheus形式のメトリクスを返します（設定時）。

TX確認ポーリング

USDT送金後、TronGrid APIを30秒間隔でポーリングし、ブロックチェーン上での確認状態を追跡します。

  • 確認済み → ステータスを confirmed に更新
  • 失敗（REVERT/OUT_OF_ENERGY）→ chain_failed
  • 30分タイムアウト → timeout（Telegram通知）

17トラブルシューティング

症状原因対処
ログインできない パスワード間違い / セッション期限切れ 正しいパスワードで再試行。.envのBK_ADMIN_PASSWORDを確認
レートが表示されない 取引所API接続失敗 / cron未実行 ログで取引所APIエラーを確認。ネットワーク疎通をチェック
USDT送金失敗 残高不足 / TRX不足 / 秘密鍵エラー ウォレットのUSDT/TRX残高を確認。TRON_WALLET_PRIVATE_KEYを確認
TX確認がタイムアウト TRONネットワーク遅延 / TronGrid API制限 TRONGRID_API_KEYを設定。TRONScanで手動確認
注文が期限切れになる タイムアウト設定が短い / 入金検知遅延 設定でタイムアウトを延長（デフォルト15分）
ビルドエラー (tsc) TypeScript型エラー npm run build のエラー出力を確認。Node.js 20+を使用
ポート3003が使用中 別プロセスが占有 .envでPORTを変更、またはnetstat -ano | findstr :3003で確認
Telegram通知が来ない BOT_TOKEN / CHAT_ID 未設定 ENABLE_NOTIFIER=true + TELEGRAM_BOT_TOKEN + TELEGRAM_STAFF_CHAT_IDを設定

20手数料自動計算・マージン安全

取引コストを下回る手数料が設定されることを防ぎ、すべての注文で確実に利益を確保するためのマージン安全機能です。

20.1 課題と解決策

課題解決策
TRONガス代・銀行振込手数料・取引所手数料を考慮せず手数料率を設定 → 赤字注文が発生するリスク 注文作成時にコストを自動見積もり、手数料率がコスト割れの場合は自動引き上げ
利益レポートが粗利のみで実質利益（コスト差引後）が不明 profit_records に total_cost / net_profit カラムを追加し、実質利益を追跡
USDT送金時のTRONガス代が記録されていない 送金成功時に transaction_costs テーブルへ自動記録

20.3 自動調整ロジック

注文作成時（買い・売り両方）に以下のロジックが実行されます：

20.4 計算式

// コスト見積
estimatedCost = tron_gas_jpy + bank_transfer_fee_jpy + (amount × exchange_fee_rate)

// 最低手数料（2つの基準の大きい方を採用）
minFeeJpy = max( estimatedCost + min_margin_jpy , amount × min_margin_rate )

// 最低手数料率
minFeeRate = minFeeJpy / amount

// 自動調整判定
if (feeRate < minFeeRate) → feeRate = minFeeRate // 自動引き上げ

// 利益レポート
net_profit = (spread_profit + fee_profit) - total_cost
